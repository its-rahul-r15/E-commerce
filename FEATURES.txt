  4. AI FEATURES (Powered by Groq API — LLaMA 3.3-70B)
================================================================================

  A. CUSTOMER AI SHOPPING ASSISTANT
  ─────────────────────────────────────────────────────────────────────────────
  • Floating chat widget (✨ gold button, bottom-right)
  • Dual-layer AI engine:
      1. Smart Rule-Based Parser (instant, works offline)
         - Extracts: category, price range, color, size from natural language
         - Handles 30+ colors, 20+ product types (Indian + Western)
         - Filler word removal: "find me", "I need", "show me" → clean keywords
      2. Groq AI Enhancement (LLaMA 3.3-70B, ~300ms response)
         - Improves accuracy for complex, mixed queries
         - Understands context and intent
         - Silent fallback if API unavailable

  • Natural Language Examples Supported:
      - "show me silk kurtas under ₹2000 in red"
      - "find blazer for office under 3000"
      - "I want a blue saree between 1500 and 5000"
      - "cotton kurta XL size"
      - "embroidered lehenga for wedding"

  • 3-Pass Search Strategy (ensures results always found):
      Pass 1: Filtered search (search + price + color + size)
      Pass 2: Keyword-only search via dedicated /search endpoint
      Pass 3: Broader word-based fallback search

  • Voice Input — speak your query in Indian English
  • Product cards shown directly inside chat (click → product page)
  • Smart "no results" messaging with helpful suggestions

  B. SELLER AI BUSINESS ADVISOR
  ─────────────────────────────────────────────────────────────────────────────
  • Built into Seller Dashboard (right sidebar widget)
  • 3 Tabs:

    [INSIGHTS TAB]
    • Real-time business analysis of seller's data
    • 4 AI-generated insights per session:
        - Revenue performance analysis
        - Pending orders urgency alerts
        - Inventory gap identification
        - Growth recommendations
    • Insight types: Success / Warning / Info / Tip (color-coded)

    [TRENDS TAB]
    • Platform-wide trending category analysis
    • Compares seller's current categories vs trending
    • 3 trend recommendations with:
        - Trend score (1-10)
        - Why it's trending now
        - Specific action to take (e.g., "Add 5 more kurtas")

    [ASK AI TAB]
    • Free-form Q&A about seller's business
    • Voice input supported
    • Example questions:
        - "What is my total revenue this month?"
        - "Which is my best selling product?"
        - "How many orders are pending?"
        - "Should I offer a discount this week?"
    • Uses actual seller data for specific, accurate answers
    • Friendly, encouraging tone in responses


================================================================================
  5. TECHNICAL HIGHLIGHTS
================================================================================

  FRONTEND
  ─────────────────────────────────────────────────────────────────────────────
  • React.js with Vite (lightning fast HMR)
  • React Router v6 — multi-page SPA with protected routes
  • Custom CSS Design System (Athenic theme variables)
  • Recharts — interactive analytics charts
  • Web Speech API — voice input for AI assistants
  • Axios with interceptors for API calls
  • Context API for global state (Auth, Cart)

  BACKEND
  ─────────────────────────────────────────────────────────────────────────────
  • Node.js + Express.js REST API
  • MongoDB + Mongoose (ODM)
  • Redis caching — product listings cached for 5 minutes
  • JWT Authentication (access token)
  • Passport.js — Google OAuth 2.0
  • Multer + Cloudinary — image upload and storage
  • MongoDB Geospatial Queries — $near for location-based shop discovery
  • Rate Limiting (express-rate-limit) — API protection
  • Role-based middleware (customer, seller, admin guards)
  • Regex-based full-text search across multiple product fields

  DATABASE DESIGN
  ─────────────────────────────────────────────────────────────────────────────
  • Users collection (roles: customer | seller | admin)
  • Shops collection (geospatial indexed, approval workflow)
  • Products collection (text + geospatial indexes, multi-image)
  • Orders collection (multi-item, status tracking)
  • Cart collection (per user, persistent)
  • Coupons collection (flexible discount rules)

  AI ARCHITECTURE
  ─────────────────────────────────────────────────────────────────────────────
  • Primary: Groq API (LLaMA 3.3-70B Versatile) — OpenAI-compatible
  • Fallback: Smart Rule-Based NLP Parser (always works, 0ms)
  • Response caching: Same query cached 5 minutes (no duplicate API calls)
  • Graceful degradation: App fully functional even without API key


================================================================================
  6. SECURITY FEATURES
================================================================================

  • JWT token authentication (stateless, secure)
  • HTTP-only cookie for token storage (XSS protection)
  • bcrypt password hashing (salt rounds: 12)
  • Route-level role guards (customer/seller/admin middleware)
  • Rate limiting on all endpoints (read: 100/min, write: 20/min)
  • Google OAuth 2.0 (no password stored for OAuth users)
  • Input validation and sanitization
  • MongoDB injection prevention via Mongoose
  • CORS configured for specific frontend origins
  • Image upload validation (type + size checks)
  • Admin-only endpoints protected at both route and controller level
